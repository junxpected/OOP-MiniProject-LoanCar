# Звіт по Ітерації 1 (Lab 34)

## Що вже працює
- Побудовано архітектурний каркас із правильним розподілом залежностей (Domain -> Application -> Infrastructure -> Console).
- Реалізовано предметну модель з інкапсульованими властивостями.
- Впроваджено поліморфізм для розрахунку запасу ходу (електромобілі та бензинові авто).
- Працює перший вертикальний зріз: авторизація користувача, перевірка стажу водіння, пошук найближчого авто та його бронювання.
- Бізнес-логіка покрита юніт-тестами.

## Артефакти в репозиторії
- `docs/vision.md` — постановка задачі.
- `docs/backlog.md` — план ітерацій.
- `docs/class-diagram.md` та `docs/sequence-diagram.md` — UML моделювання.
- `.github/workflows/dotnet.yml` — налаштований CI.
- `README.md` — опис проєкту.

## Сценарії для розширення на Lab 35
1. Завершення оренди (зміна координат авто на нові та повернення статусу "Доступно").
2. Впровадження файлового збереження даних (JSON) замість In-Memory репозиторіїв.
3. Додавання нового статусу `NeedsMaintenance` для авто з низьким запасом ходу.

## Ризики або невизначеності
- Наразі дані губляться після закриття консолі. Це буде вирішено на наступній ітерації через серіалізацію.
- Розрахунок відстані ведеться за звичайною геометрією, без урахування реальних доріг.

## Класи/Інтерфейси, підготовлені під розширення
- `Vehicle`: абстрактний клас, який дозволяє легко додати гібридні автомобілі або водневий транспорт.
- `IUserRepository` та `IVehicleRepository`: дозволять без зміни бізнес-логіки підключити реальну базу даних.
