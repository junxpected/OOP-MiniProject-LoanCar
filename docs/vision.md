# Vision: NextGen CarSharing

## Опис проблеми
Створення сучасної системи каршерінгу, яка враховує індивідуальні характеристики користувачів та автомобілів. Існуючі базові системи не обмежують доступ до складних або дорогих авто для новачків і не дають зручного розуміння запасу ходу (у відсотках чи літрах орієнтуватися важко). Наш застосунок вирішує це, додаючи систему допусків за стажем та автоматичний розрахунок запасу ходу в кілометрах для будь-якого типу двигуна.

## Цільові користувачі
Користувачі послуг каршерінгу (водії з різним рівнем стажу), яким потрібен транспорт для специфічних потреб (економ для міста, вантажне для переїздів, преміум для зустрічей).

## Основні сценарії використання
1. **Авторизація:** Користувач входить у систему під своїм іменем, система завантажує його профіль (стаж водіння, наявність активної оренди).
2. **Безпечний пошук:** Користувач обирає клас авто (Economy, Comfort, Premium, Cargo). Система перевіряє, чи має користувач достатній стаж (наприклад, > 2 років для Premium/Cargo).
3. **Вибір та бронювання:** Система показує найближчі доступні авто обраного класу, розраховує відстань до них та їхній запас ходу в кілометрах (за допомогою поліморфізму для електро- та бензинових авто). Користувач бронює авто.

## Нефункціональні вимоги
1. **Чиста архітектура:** Чітке розділення коду на Domain (сутність автомобіля/користувача), Application (сервіси оренди) та Infrastructure (збереження даних).
2. **Читабельність та ООП:** Використання поліморфізму для різних типів авто та інкапсуляція бізнес-правил (наприклад, перевірка стажу) всередині класів.
3. **Обробка помилок:** Використання патерну `Result` замість винятків (Exceptions) для керування бізнес-логікою.

## Обмеження Ітерації 1 (Lab 34)
- Дані користувачів та авто зберігаються в пам'яті (In-Memory), без підключення реальної бази даних.
- Інтерфейс реалізовано виключно через консоль.
- Локація обчислюється через просту математичну відстань (без інтеграції з Google Maps).
